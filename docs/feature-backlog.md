## 指令
1. 使用 @ + 文件路径，如@cmd/main.go 将文件内容与元数据附带在用户输入信息内，但是不要渲染给用户，前端添加一个 + 按钮允许鼠标点击添加
2. 使用 / + 命令/skill name 添加一些指令功能，如，压缩记忆
    - 可预设的特殊prompt（比方说生成Agents.md），要可配置，可以在设置页面添加一个模块专门干这个
    - 压缩记忆（上下文）功能，同时添加自动压缩上下文阈值
    - 显式要求agent使用指定skill，跳过agent自己调用查看skill功能，直接把skill内容附在用户信息内
    - goal 制定一个目标，要求agent完成，在agent结束一轮对话时自动添加要求检查和显式结束（如调用一个goal完成的工具）并提供实现结果
    - plan 根据用户上下文，制定一个计划，添加计划相关的工具，如添加计划，完成计划（这个工具需要模型提供完成理由/成果，可以让用户看到，而不是只打一个对勾），计划要在右侧侧边栏（待定，可以讨论如何可视化）可视，模型必须关闭所有计划才能结束对话，否则自动回复要求检查与完成计划的prompt
